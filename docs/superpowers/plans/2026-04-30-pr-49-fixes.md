## Task 2: Spec correctness — Ppse priority nibble

**File:** `shared/src/commonMain/kotlin/io/github/a7asoft/nfcemv/extract/Ppse.kt`

Find the priority decoding line (likely around line 23 per emv-expert flag — may be inside a `parseEntry` helper or similar). Currently:
```kotlin
val priority = priorityBytes.firstOrNull()?.toInt()?.and(0xFF)
```

Change to (per EMV Book 1 §12.2.3 — only low nibble carries the priority value 1..15; b8 is the cardholder-confirmation flag, b5–b7 are RFU):
```kotlin
// why: per EMV Book 1 §12.2.3, only the low nibble of the Application
// Priority Indicator carries the priority value (1..15). Bit b8 is the
// "cardholder confirmation supported" flag and b5–b7 are RFU. A card
// sending 0x81 means priority 1 with confirmation required, NOT priority
// 0x81 = 129. Decoding the full byte ranks confirmation-requiring cards
// last instead of first. Treat 0x00 as "no priority assigned" (null).
val priority = priorityBytes.firstOrNull()?.toInt()?.and(0x0F)?.takeIf { it != 0 }
```

Update KDoc on `PpseEntry.priority` to clarify:
```kotlin
/**
 * Application Priority Indicator low nibble (1..15) per EMV Book 1
 * §12.2.3, or `null` if the application has no priority assigned (the
 * card omitted tag `87`, sent priority 0, or sent only RFU/flag bits).
 * Lower value wins; absent priority sorts last via
 * [ContactlessReader]'s `minByOrNull { priority ?: Int.MAX_VALUE }`.
 */
val priority: Int?
```

**File:** `shared/src/commonMain/kotlin/io/github/a7asoft/nfcemv/extract/Gpo.kt`

### Step 6a — Convert `Gpo` from `data class` to `class`

Current:
```kotlin

```

After (mirrors `Tlv.Primitive` pattern from #1):
```kotlin
public class Gpo internal constructor(
    aip: ByteArray,
    public val afl: Afl,
) {
    private val storedAip: ByteArray = aip.copyOf()

    /**
     * Application Interchange Profile (EMV Book 3 Annex C1) — a 2-byte
     * capability bitmap. Returns a fresh copy per access so callers
     * cannot mutate the underlying bytes.
     */
    public val applicationInterchangeProfile: ByteArray
        get() = storedAip.copyOf()

    override fun equals(other: Any?): Boolean {
        if (this === other) return true
        if (other !is Gpo) return false
        if (!storedAip.contentEquals(other.storedAip)) return false
        if (afl != other.afl) return false
        return true
    }

    override fun hashCode(): Int {
        var result = storedAip.contentHashCode()
        result = 31 * result + afl.hashCode()
        return result
    }

    override fun toString(): String =
        "Gpo(aip=${storedAip.size} bytes, afl=$afl)"

    public companion object
}
```

The `toString` deliberately prints byte-COUNT not bytes (AIP is 2 bytes, not PCI, but the pattern guards against future ByteArray fields on similar types).

## Task 9: `readAllRecords` silent-skip — pin behavior + `// why:`

**File:** `android/reader/src/main/kotlin/io/github/a7asoft/nfcemv/reader/ContactlessReader.kt`

Find `readAllRecords`. Currently silently skips non-9000 READ RECORD responses. Decision: KEEP the skip behavior (pragmatic for cards with sparse AFLs), but:

1. Add `// why:` comment explaining the trade-off.
2. Add a test pinning the behavior so future contributors don't accidentally change it.

Code change:
```kotlin
private fun readAllRecords(afl: Afl): List<ByteArray> {
    val collected = mutableListOf<ByteArray>()
    afl.entries.forEach { entry ->
        for (record in entry.firstRecord..entry.lastRecord) {
            val response = transport.transceive(ApduCommands.readRecord(record, entry.sfi))
            // why: a non-9000 READ RECORD is silently skipped rather than aborting
            // the whole flow. Real cards sometimes advertise records in their AFL
            // that aren't readable (file-not-activated, end-of-file). EmvParser
            // surfaces MissingRequiredTag downstream if essential data was in
            // a skipped record. Trade-off accepted for v0.2.0; see docs/threat-
            // model.md and follow-up issue if this turns out to mask real bugs.
            if (ApduCommands.isSuccess(response)) {
                collected.add(ApduCommands.dataField(response))
            }
        }
    }
    return collected
}
```
